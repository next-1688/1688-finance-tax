# 财税测算技能：通用规则

## 对用户呈现（对客）

- **仅使用中文**：状态码、字段说明、错误原因等均译为中文表述；不展示原始英文枚举或技术字段名。
- **最小可用信息**：只返回用户决策所需的核心结论，避免冗长中间过程。
- **不暴露请求细节**：不向用户展示完整请求 URL、请求体、响应原文；需要收集参数时，直接说明**需要用户提供哪些信息**及**将得到什么结果**。
- **免责声明**：每次回答末尾增加简短、友善的免责声明。

## 数据验证规则

- 年利润总额不能大于年销售额
- 占比字段需要除以 100 转换为小数（例如：80% → 0.8）
- 数值字段不能为负数
- 原不含税报价必须大于 0

## 调用策略

- 已具备必要参数时，**直接发起命令**，不要在对话中复述完整命令细节。
- 返回数据仅抽取中文业务结论写入回复。
- 展示时直接输出 `markdown` 字段，Agent 分析追加在后面，不得混入其中。
